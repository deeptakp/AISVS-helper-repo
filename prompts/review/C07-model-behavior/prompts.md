### C7-REVIEW-01: Review Output Validation Controls

**Objective**
- You are reviewing an LLM application's output handling for AISVS C7.1 compliance.

**Required Controls**
- Use AISVS controls declared in file header and prompt scope.

**Execution Steps**
- C7.1.1 — Is every model output validated against a declared schema before being returned to callers?
- C7.1.2 — Are output length limits and termination controls enforced?
- C7.3.1 — Is an automated safety classifier applied to every response?
- C7.3.2 — Is there a system prompt leak detector applied to outputs?
- C7.3.3 — Is model-generated output prevented from triggering outbound requests?
- C7.3.4 (L3) — Are outputs checked for hidden/encoded content (homoglyphs, zero-width chars, metadata)?
- For each FAIL: state gap, control ID, and remediation with example code.

**Expected Artifacts**
- Evidence logs, control mapping, and remediation notes for each finding/gap.

**Pass/Fail Rule**
- Mark PASS only when all required controls in scope are satisfied with evidence; otherwise mark FAIL and provide remediation.

**Source Prompt**
```text
You are reviewing an LLM application's output handling for AISVS C7.1 compliance.

Check each item:
[ ] C7.1.1 — Is every model output validated against a declared schema before being returned to callers?
[ ] C7.1.2 — Are output length limits and termination controls enforced?
[ ] C7.3.1 — Is an automated safety classifier applied to every response?
[ ] C7.3.2 — Is there a system prompt leak detector applied to outputs?
[ ] C7.3.3 — Is model-generated output prevented from triggering outbound requests?
[ ] C7.3.4 (L3) — Are outputs checked for hidden/encoded content (homoglyphs, zero-width chars, metadata)?

For each FAIL: state gap, control ID, and remediation with example code.
```


### C7-REVIEW-02: Review RAG Attribution and Hallucination Controls

**Objective**
- You are reviewing a RAG-based AI application for AISVS C7.2 and C7.4 compliance.

**Required Controls**
- Use AISVS controls declared in file header and prompt scope.

**Execution Steps**
- C7.2.1 — Does the system assess reliability of generated answers using a confidence estimation method?
- C7.2.2 — Does the system block or fall back if confidence drops below a defined threshold?
- C7.2.3 (L3) — Are high-risk responses routed to an additional verification step?
- C7.4.1 — Do RAG responses include attribution to source documents?
- C7.4.2 — Are attributions derived from retrieval metadata (not generated by the model)?
- C7.4.3 — Can claims in RAG responses be traced to retrieved chunks?
- C7.4.4 (L3) — Is AI-generated media watermarked?
